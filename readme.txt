【配置说明】
・环境要求：JRE1.7以上
・网络要求：
服务端、客户端、管理端要进行网络通信，为保证安全，将通讯地址封装在了程序内部。由于测试环境限制，封装的通讯地址都指向本机，所以使用的时候三个程序需要运行在同一机器上。
・文件要求：
程序会进行冷启动，管理端配置文件为config.dat，如果没有会自动生成。客户端配置文件为groups.dat，没有程序会强制退出。groups.dat可以由管理端添加组织生成。管理端还需要1.jpg文件支持。
・初次运行顺序建议
启动服务端->打开管理端->添加组织信息->关闭管理端（保存数据）->启动客户端
【程序说明】
・server.jar
服务器端，用于数据交互及数据存储。
・admin.jar
管理端，用于数据管理、信息查看等。
・user.jar
客户端，用于时长录入，组织内志愿者管理等。
